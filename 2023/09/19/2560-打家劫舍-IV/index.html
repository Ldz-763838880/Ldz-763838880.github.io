2560-打家劫舍 IV

1 题目描述

沿街有一排连续的房屋。每间房屋内都藏有一定的现金。现在有一位小偷计划从这些房屋中窃取现金。

由于相邻的房屋装有相互连通的防盗系统，所以小偷 不会窃取相邻的房屋 。

小偷的 窃取能力 定义为他在窃取过程中能从单间房屋中窃取的 最大金额 。

给你一个整数数组 nums 表示每间房屋存放的现金金额。形式上，从左起第 i 间房屋中放有 nums[i] 美元。

另给你一个整数 k ，表示窃贼将会窃取的 最少 房屋数。小偷总能窃取至少 k 间房屋。

返回小偷的 最小 窃取能力。

示例

输入：nums = [2,3,5,9], k = 2
输出：5
解释：
小偷窃取至少 2 间房屋，共有 3 种方式：
- 窃取下标 0 和 2 处的房屋，窃取能力为 max(nums[0], nums[2]) = 5 。
- 窃取下标 0 和 3 处的房屋，窃取能力为 max(nums[0], nums[3]) = 9 。
- 窃取下标 1 和 3 处的房屋，窃取能力为 max(nums[1], nums[3]) = 9 。
因此，返回 min(5, 9, 9) = 5 。

2 解题思路

  • 思路链接

  • 二分 + DP

    • mx 表示最大窃取金额，定义 f[i]表示从 nums[0]到 nums[i]中偷金额不超过mx 的房屋，最多能偷多少间房屋
    • 使用二个遍历f0,f1表示当前i房间是否被偷，初始状态，f0, f1 = 0
    • 状态转移方程：
      • i房间不偷
        • f0 = f1
      • i房间偷
        • f1 = max（f1， f0 + 1)
    • 最后二分查找[0, max(nums)]，mx为二分值，找到满足k的第一个mx,而且mx必定是在nums里面

  • 二分 + 贪心

    • mx 表示最大窃取金额，统计当前满足mx时偷取的最大房屋数量，可以证明遍历nums可以偷的就直接偷
    • 最后二分查找[0, max(nums)]，mx为二分值，找到满足k的第一个mx,而且mx必定是在nums里面

3 复杂度分析

  • 时间复杂度
    • 二分 + DP/贪心
      • O(nlog(U))
      • 遍历一遍矩阵 + 二分查找
  • 空间复杂度
    • 二分 + DP/贪心
      • O(1)
      • 常规变量

4 代码

// 二分 + DP/贪心 
class Solution:
    def minCapability(self, nums: List[int], k: int) -> int:
        # 二分 + DP
        def solve(mx: int) -> int:
            f0 = f1 = 0
            for num in nums:
                if num > mx:
                    f0 = f1
                else:
                    f0, f1 = f1, max(f1, f0 + 1)
            return f1
        return bisect_left(range(max(nums)), k, key = solve)

        # 二分 + 贪心
        def solve(mx: int) -> int:
            cnt = i = 0
            while i < len(nums):
                if nums[i] > mx:
                    i += 1
                else:
                    cnt += 1
                    i += 2
            return cnt

        return bisect_left(range(max(nums)), k, key = solve)
